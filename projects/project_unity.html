Unity C# 태스크 개발

Unity / C# VR 실험 개발 SW 파이프라인 설계

2023 – 2024 | 서울대 인지발달신경과학 연구실

배경

석사 연구와 사수의 박사 연구 데이터 수집을 위해 VR 기반 공간 내비게이션 실험 task 파이프라인을 처음부터 개발해야 했습니다. C#을 전혀 모르는 상태였고, 연구실에 Unity를 다뤄본 사람이 없었습니다. 6개월 안에 완성해 실제 7T fMRI 실험에 투입해야 하는 상황이었습니다.

구체적인 실험 패러다임은 미출판 연구로 공개할 수 없습니다.

문제

가장 어려웠던 것은 확정되지 않은 요구사항 아래서 계속 만들어야 한다는 것이었습니다. 인지과학 실험 특성상 가설이 구체화될수록 패러다임이 업데이트됐고, 각 가안을 직접 구현해 파일럿을 돌려봐야만 문제를 발견할 수 있었습니다. 수정이 잦아 처음부터 다시 만든 경우도 여러 번이었습니다.

구현

전역 상태 관리

다단계로 구성된 실험 흐름 전체를 단일 SbjManager 클래스의 static 변수로 관리했습니다. 현재 task, phase, 환경, trial, run 상태를 전역에서 추적해 씬 전환 시에도 데이터 일관성을 유지했습니다.

적응형 종료 조건

피험자별 수행 오차를 누적 추적해 기준값 이하로 수렴할 때 단계가 종료되는 적응형 구조를 설계했습니다. 최소/최대 시행 수를 설정해 예외 상황에서 무한 루프를 방지하는 안전장치를 구현했습니다.

실시간 데이터 로깅

StreamWriter를 활용해 행동 데이터, 타임스탬프, 이벤트 로그를 용도별 .txt 파일로 분리해 실시간 기록했습니다. 파일 핸들을 전역에서 단일 관리해 씬 전환 시 데이터 손실을 방지하는 구조를 구축했습니다.

피험자별 카운터밸런싱

피험자 번호 기반으로 조건 노출 순서를 자동 배정하는 로직을 구현해 실험자가 수동으로 개입하지 않아도 되는 구조를 만들었습니다.

결과

6개월 만에 파이프라인 완성. 7T fMRI 환경에서 33명의 데이터를 수집했습니다. 수집된 행동 로그는 MATLAB 기반 분석 파이프라인의 입력 데이터로 직접 연결됩니다.

실무 연결

SW 개발 경험이 없는 상태에서 요구사항이 계속 바뀌는 환경에 대응하는 구조를 직접 설계하고 완성한 경험입니다. 데이터가 이후 분석에서 어떻게 쓰일지를 먼저 따진 뒤 수집 파이프라인을 설계하는 방식은 서비스 데이터 파이프라인 설계에도 동일하게 적용됩니다.
